# 测试数据生成 Skill

## 激活场景
- 用例设计完成后需要填充测试数据
- 用户要求生成某类型字段的测试数据集
- 安全测试需要攻击性Payload数据
- 接口测试需要各类边界数据

## 通用字段数据库

### 用户认证类数据

```yaml
手机号:
  有效数据:
    - "13812345678"    # 标准11位手机号
    - "15900000001"    # 最小有效号段
    - "19999999999"    # 最大有效号段
  边界数据:
    - "1381234567"     # 10位（下边界-1）
    - "138123456789"   # 12位（上边界+1）
    - "00000000000"    # 全零
    - "11111111111"    # 全相同数字
  无效数据:
    - ""               # 空字符串
    - " "              # 纯空格
    - "1381234567a"    # 含字母
    - "+8613812345678" # 国际格式
    - "abc"            # 纯字母
  安全数据:
    - "'; DROP TABLE users; --"   # SQL注入
    - "<script>alert(1)</script>" # XSS

密码:
  有效数据:
    - "Test@123456"    # 大小写+数字+特殊字符
    - "Aa1!Bb2@Cc3#"  # 复杂密码
  边界数据:
    - "Aa1!567"        # 最小长度（8位）
    - "Aa1!" + "x"×60 # 最大长度（64位）
    - "aa1!aaaa"       # 缺大写字母
    - "AA1!AAAA"       # 缺小写字母
    - "AaAaAa!!"       # 缺数字
    - "AaBb1234"       # 缺特殊字符
  无效数据:
    - "12345678"       # 纯数字
    - "password"       # 纯字母（常见弱密码）
    - "Aa1!567"[:-1]   # 7位（下边界-1）
    - ""               # 空密码

邮箱:
  有效数据:
    - "user@example.com"
    - "user.name+tag@domain.co.uk"
    - "user123@sub.domain.com"
  无效数据:
    - "userexample.com"    # 缺@
    - "user@"              # 缺域名
    - "@example.com"       # 缺用户名
    - "user @example.com"  # 含空格
    - "user@example"       # 缺顶级域
```

### 字符串类数据

```yaml
通用文本字段:
  空值系列:
    - ""                   # 空字符串
    - " "                  # 单空格
    - "   "                # 多空格
    - "\t"                 # Tab字符
    - "\n"                 # 换行符
  特殊字符系列:
    - "!@#$%^&*()"         # 常见特殊字符
    - "中文测试数据"         # 中文字符
    - "日本語テスト"         # 日文字符
    - "🎉🔥💯"              # Emoji字符
    - "اختبار"             # 阿拉伯语（RTL）
  长度边界系列:
    - "a"                  # 1个字符
    - "a" × 255            # 255字符（常见DB限制）
    - "a" × 256            # 256字符（越界）
    - "a" × 65535          # 超长字符串

    - "' OR '1'='1"        # SQL注入经典Payload
    - "'; DROP TABLE--"    # SQL删表注入
    - "<script>alert(document.cookie)</script>"  # XSS
    - "{{7*7}}"            # 模板注入（SSTI）

    - "%00"                # Null字节注入
```

### 日期时间类数据

```yaml
日期(YYYY-MM-DD):
  有效数据:
    - "2024-01-01"    # 年初
    - "2024-12-31"    # 年末
    - "2024-02-29"    # 闰年2月29日
    - "2024-06-15"    # 普通日期
  边界数据:
    - "2024-02-28"    # 非闰年2月末
    - "2023-02-29"    # 非闰年2月29（无效）
    - "2024-04-31"    # 4月没有31日（无效）
    - "2024-13-01"    # 13月（无效）
    - "2024-00-01"    # 0月（无效）
  无效数据:
    - "2024/01/01"    # 格式错误（斜杠）
    - "01-01-2024"    # 格式错误（月日年）
    - "abc"           # 非日期
    - ""              # 空值
    - "9999-12-31"    # 极远未来

时间范围（开始时间/结束时间）:
  有效组合:
    start: "2024-01-01", end: "2024-12-31"
  边界组合:
    - start == end（同一天）
    - end = start + 1天（最小范围）
  无效组合:
    - start > end（结束早于开始）
    - start == end == null（两者均空）
```

### 文件上传类数据

```yaml
图片上传测试:
  有效文件:
    - 标准JPG（100KB）：content_type=image/jpeg
    - 标准PNG（50KB）：content_type=image/png
    - 标准GIF（动图，200KB）
    - 标准WebP（80KB）
  边界文件:
    - 0字节空图片：应提示文件损坏
    - 1字节文件：应提示无效图片
    - 9.9MB图片：接近上限（假设10MB限制）
    - 10MB图片：恰好达到限制
    - 10.1MB图片：超出限制，应拒绝
  无效文件:
    - PHP文件改扩展名为.jpg：安全测试
    - EXE文件改扩展名为.png：安全测试
    - 包含EXIF敏感信息的图片：隐私测试
    - SVG文件含JS代码：XSS测试
```

## 数据生成规则说明

生成测试数据时遵循以下优先级顺序：

```
P0（必须覆盖）：
  1. 最典型的有效值（Happy Path数据）
  2. 下边界值、上边界值
  3. 空值/null/零值

P1（应该覆盖）：
  4. 边界-1、边界+1
  5. 无效等价类代表值
  6. 中间值

P2（建议覆盖）：
  7. 特殊字符
  8. 超长字符串
  9. 类型错误（数字传字符串）

P3（有时间则覆盖）：
  10. SQL注入Payload
  11. XSS Payload
  12. Unicode边缘字符
```

## 数据输出格式

```markdown
**测试数据集 - [字段名/场景名]**

| 数据类别 | 具体数据值 | 说明 | 预期行为 |
|---------|-----------|------|---------|
| 有效数据 | testuser@qq.com | 标准邮箱格式 | 接受，正常处理 |
| 边界数据 | a@b.cc | 最短有效邮箱 | 接受 |
| 无效数据 | notanemail | 缺少@和域名 | 拒绝，提示格式错误 |
| 安全数据 | "><script>alert(1)</script>@x.com | XSS | 拒绝或转义输出 |
```
